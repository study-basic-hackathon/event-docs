## 背景・目的

「プログラミング初学者の会」では、年に2-3回、1ヶ月かけて実施するハッカソンを開催しています。初めて会った3人1チームで、サービスの内容検討から実装までを1ヶ月で実施する本格的なイベントです。

しかし、初心者の方にとってはハードルが高く、特にチームで開発するためのGitスキルが不足していることが課題となっています。そこで、ハッカソンに参加できる人を増やすため、Git基礎セミナーを開催することになりました。

## 実施内容（詳細）

### 1. オープニング（13:00-13:15）
- イベントの目的と進め方の説明
- 参加者の簡単な自己紹介（名前、Gitの経験レベル）
- 質問しやすい雰囲気づくり

### 2. Gitとは？基礎講義（13:15-13:45）
**学習目標**:
- バージョン管理の必要性を理解する
- Gitの基本的な仕組みを理解する
- チーム開発でGitがどう役立つかを理解する

**内容**:
- バージョン管理システムとは
- Gitの歴史と特徴（分散型バージョン管理）
- GitとGitHubの違い
- リポジトリ、コミット、ブランチの概念
- 実際の開発フローのイメージ

### 3. 環境構築・初期設定（13:45-14:30）
**学習目標**:
- 自分のPCでGitを使えるようにする
- GitHubアカウントを準備する

**内容**:
- Git インストール状況の確認
  - Windows: Git for Windows
  - Mac: Xcode Command Line Tools / Homebrew
- GitHubアカウントの作成/ログイン確認
- Git初期設定
  ```bash
  git config --global user.name "Your Name"
  git config --global user.email "your.email@example.com"
  ```
- （時間があれば）SSHキーの設定

### 4. ハンズオン①：基本操作（14:45-15:30）
**学習目標**:
- ローカルリポジトリを作成できる
- ファイルの変更をコミットできる
- コミット履歴を確認できる

**実習内容**:
1. 新しいディレクトリを作成
2. `git init` でリポジトリ初期化
3. README.mdファイルを作成
4. `git add` でステージング
5. `git commit` でコミット
6. ファイルを編集して再度コミット
7. `git log` で履歴確認
8. `git status` でファイル状態確認

### 5. ハンズオン②：GitHub連携（15:30-16:15）
**学習目標**:
- リモートリポジトリの概念を理解する
- ローカルの変更をGitHubにプッシュできる
- GitHubから変更を取得できる

**実習内容**:
1. GitHubで新しいリポジトリを作成
2. ローカルリポジトリとリモートリポジトリを接続
   ```bash
   git remote add origin <repository-url>
   ```
3. `git push` でリモートにプッシュ
4. GitHubのWebページで変更を確認
5. GitHubのWeb画面から直接ファイルを編集
6. `git pull` でローカルに反映

### 6. ハンズオン③：ブランチ操作（16:15-16:50）
**学習目標**:
- ブランチの概念と必要性を理解する
- ブランチを作成して切り替えられる
- ブランチをマージできる

**実習内容**:
1. ブランチとは何か（feature開発、複数人での開発）
2. `git branch` で新しいブランチを作成
3. `git checkout` / `git switch` でブランチ切り替え
4. 新しいブランチで変更をコミット
5. mainブランチに戻る
6. `git merge` でブランチをマージ
7. （時間があれば）簡単なコンフリクトの解決

### 7. 実践演習：チーム開発体験（16:40-16:55）
**学習目標**:
- 実際のチーム開発フローを体験する
- プルリクエストを作成できる
- コラボレーション開発の雰囲気を味わう

**実習内容**:
1. 3-4人のグループに分かれる
2. 共通リポジトリに自己紹介ページを追加
3. 各自がブランチを作成
4. 自己紹介ファイルを作成・コミット
5. プルリクエストを作成
6. （時間があれば）他の人のプルリクエストをレビュー

**使用リポジトリ**:
- 事前に `community-site` リポジトリを準備
- 参加者をコラボレーターに招待
- `members/` ディレクトリを用意

**演習の流れ**:
```bash
# 1. リポジトリをクローン
git clone [repository-url]

# 2. ブランチ作成
git switch -c add-member-[自分の名前]

# 3. 自己紹介ファイル作成
# members/[名前].md

# 4. コミット
git add .
git commit -m "add: [名前]の自己紹介"

# 5. プッシュ
git push -u origin add-member-[自分の名前]

# 6. GitHubでプルリクエスト作成
```

## 想定される質問・トラブル対応

### よくある質問
1. Q: GitとGitHubの違いは？
   - A: Gitはバージョン管理システム、GitHubはGitリポジトリをホスティングするサービス

2. Q: コミットメッセージはどう書けばいい？
   - A: 変更内容が分かるように具体的に書く（例：「ログイン機能を追加」）

3. Q: pushできない時は？
   - A: pullしてから再度pushする、リモートの変更を取り込む必要がある

### トラブルシューティング
- Git インストールされていない
  - その場で公式サイトからダウンロード
- GitHubアカウントがない
  - その場で作成（メール認証に時間がかかる可能性）
- 認証エラー
  - HTTPS接続に切り替える
  - Personal Access Tokenの設定を案内
